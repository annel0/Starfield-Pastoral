# 实用设施注册与实现流程（草案）

> 目标：规范“可交互处理型方块”的完整实现与注册流程，适配现有的蟹笼与树液采集器体系，并统一 Jade 显示规范。

## 1. 定义与范围
**实用设施**：玩家放置后，可在“投入物/条件满足 → 工作 → 产物就绪 → 收取”循环中运作的功能性方块。典型特征：
- 右键交互（含投入物/启动/收获）
- 具备工作态表现（模型/动画/发光/浮动/粒子等）
- 产物就绪提示（仅就绪时显示气泡 + 物品图标）
- Jade 提示覆盖所有阶段
- Jade 行内必须带物品图标（投入物/产物），风格与树液采集器一致

示例：蟹笼、树液采集器、熔炉类工作台、加工器等。

---

## 2. 统一实现流程（Checklist）

### 2.1 设计阶段（必做）
- **交互模型**：
  - 是否需要投入物（鱼饵/燃料/原料）
  - 是否允许无物启动（当前仅树液采集器；后续如有新增需显式注明）
  - 是否有一次性产物或可循环产物
- **时间模型**：
  - 一律使用本模组时间体系（StardewTimeManager）
  - 是否受季节/天气/时间/群系影响
- **产物模型**：
  - 产物列表/权重/品质规则（Jade 必须规范显示产物）
  - 可叠加效果（如高级饵提升品质、野生饵双倍等）
- **视觉表现**：
  - 工作态模型/动画/光照（默认沿用小桶动画；若无自定义说明则统一使用）
  - 就绪态气泡与物品图标（仅就绪时显示）
  - 破坏粒子：必须生成 16x16 的 `*_particle.png` 并绑定到模型 `particle`
  - 光影兼容策略（ENTITYBLOCK_ANIMATED 或局部动画）需在设计阶段确认

### 2.2 代码实现阶段（必做）
> 按顺序建议实现：BlockEntity → Block → Renderer → Jade → 注册 → 资源

1) **BlockEntity（逻辑核心）**
- 存储：投入物、产物、工作状态、时间戳/计时器
- 服务端 tick：
  - 定时检查（每日/分钟）
  - 计算是否就绪
  - 状态变化时同步客户端
- Debug 加速：接入 NUMPAD7 快速推进（统一通道）
- 存读档：保存投入物/产物/计时器/状态
- 提供 API：`hasInput()`、`isReady()`、`getProduct()`、`setInput()`、`clearProduct()` 等

2) **Block（交互与物理行为）**
- 放置逻辑：
  - 环境约束（树干/水下/地形）
  - 方向/状态属性
- 交互逻辑：
  - 右键投入物
  - 右键收获
  - 交互失败的提示行为（不吞交互或返回 PASS）
- 方块状态属性：
  - `WORKING`、`READY`（可选）
  - `WATERLOGGED`、`FACING` 等

3) **渲染（BlockEntityRenderer）**
- 基础模型渲染（若使用 `ENTITYBLOCK_ANIMATED`）
- 工作态：浮动/发光/替换模型/粒子
- 就绪态：气泡 + 物品图标
- 默认动画：若未指定，直接复用小桶动画参数

4) **Jade 显示**
- `IServerDataProvider`：发送必要状态
- `IBlockComponentProvider`：客户端展示
- 统一阶段显示规范（见第 3 节）
- 必须用图标行显示投入物/产物（样式与树液采集器一致）

5) **注册**
- Block 注册
- BlockEntity 注册
- BlockItem 注册
- Renderer 注册
- Jade Plugin 注册

6) **资源**
- Blockstates / Model / Texture
- Lang 文本
- 破坏粒子贴图：`textures/block/utility/*_particle.png`
- Model 中 `textures.particle` 必须指向对应 `*_particle`

## 2.3 原版逻辑对齐要点（建议保留）
> 这是从原版机制抽象出来的通用机理，不强制，但建议统一到模板里。

- **统一机器状态字段**：
  - `heldObject`（产物）
  - `minutesUntilReady`（剩余时间）
  - `readyForHarvest`（就绪标记）
  - `lastInputItem`（最近投入物）
- **通用“放入 → 产出”流程**：
  - 支持输入校验（物品类型/数量/标签）
  - 支持额外消耗（除主输入外的附加材料）
  - 支持“加载/工作/完成”特效（声音/粒子/动画）
- **工作态表现**：
  - 允许“工作中发光”“切换模型帧/索引”“抖动/浮动”
  - 默认动画基准：未定义则使用小桶工作动画
  - Debug 加速：NUMPAD7 必须可推进工作进度
- **例外逻辑**：
  - 蟹笼与树液采集器属于“有额外日更/环境逻辑”的特例

## 3. Jade 显示规范（阶段定义）

> 每个实用设施至少覆盖以下 3 个阶段：

### Stage 1：未投入 / 未启动
**判定条件**：无投入物（或条件不足）且无产物
**显示内容**：
- 主要提示：投入物/条件信息（带物品图标）
- 示例（蟹笼）：`鱼饵：无`
- 颜色建议：白色/灰色

### Stage 2：工作中
**判定条件**：已投入且未就绪（或正在处理）
**显示内容**：
- 投入物名称（若有，带物品图标）
- 工作提示或剩余时间
- 示例（蟹笼）：`鱼饵：XXX` + `明天再来看看……`
- 示例（树液采集器）：`产物：XXX` + `剩余X天Y小时`
- 颜色建议：信息白色，提示灰色

### Stage 3：已就绪
**判定条件**：产物存在且可收获
**显示内容**：
- 产物名称（含品质显示，带物品图标）
- 收获提示（绿色加粗）
- 示例：`产物：XXX` + `已完成 右键获得产物`
- 颜色建议：产物白色，提示绿色粗体

> 规则补充：阶段判定不设例外；产物显示为强制项；气泡仅在就绪阶段显示。

## 4. Jade NBT 数据规范（建议）

### 通用字段
- `ready`：是否就绪
- `productItem`：产物 ID（可选）
- `inputItem`：投入物 ID（可选）
- `remainingDays` / `remainingHours`：剩余时间（可选）
- `productQuality`：产物品质（可选）

### 蟹笼示例
- `hasBait`：是否有鱼饵
- `baitItem`：鱼饵 ID
- `ready`：是否就绪
- `productItem`：产物 ID

### 树液采集器示例
- `ready`：是否就绪
- `item`：产物 ID
- `days` / `hours`：剩余时间

## 7. 小桶（Keg）落地计划（当前执行）

### 7.1 逻辑与输入输出
- **输入 → 产物规则**：按 [KEG_ITEM_REGISTRATION_PLAN.md](KEG_ITEM_REGISTRATION_PLAN.md) 的映射逐项实现（水果→果酒、蔬菜→果汁、特殊配方）。
- **计时模型**：基于 `StardewTimeManager` 计时；产物预先生成，仅到点变为就绪。
- **咖啡豆特殊规则**：一次消耗 5 个咖啡豆，产出 1 杯咖啡。
- **品质**：产物默认普通品质（与原版一致，后续可拓展陈酿系统）。

### 7.2 表现与反馈
- **工作态动画**：方块本体上下浮动（“史莱姆式”弹动），仅工作中启用。
- **粒子**：工作中持续冒出轻微烟雾粒子（每个 utility 工作时都有粒子，先从小桶落地）。
- **放入音效**：放入原料触发一次装载音效（原版 `Object.cs` 的 `PlaceInMachine`/`PlayMachineEffect` 流程）。
- **就绪提示**：气泡 + 产物图标。
- **破坏粒子**：独立 16x16 粒子贴图（从原贴图自动取色）。

### 7.3 Jade 显示
- Stage 1：`投入物：无`
- Stage 2：`投入物：XXX` + `剩余X天Y小时`
- Stage 3：`产物：XXX` + `已完成 右键获得产物`
